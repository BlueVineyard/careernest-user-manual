Official User Manual · Version 1.17.4

CareerNest

A standalone WordPress job-portal plugin that turns any WordPress site into a fully-featured careers platform - job postings, employer accounts, applicant profiles, and applications - built entirely on WordPress core APIs with zero third-party dependencies.

📘 Last updated: 18 May 2026 🏷️ Documents version 1.17.4 👤 By Rohan T George · Blue Vineyard

This manual walks site owners, hiring teams, and developers through every aspect of CareerNest - from a five-minute first install to extending the plugin with custom theme templates. Every setting, shortcode, hook, and FAQ is documented in full. Use the sidebar (or the search box) to jump straight to what you need.

Get Started Browse Features

Introduction

CareerNest is a self-contained job-portal plugin for WordPress. It registers four custom post types (jobs, employers, applicants, applications), three custom user roles, and a complete set of public-facing pages - and ties them together with admin dashboards, approval workflows, email notifications, and a configurable job board.

Unlike most job-portal plugins, CareerNest depends on nothing but WordPress core APIs. No external services are required to run it, no premium add-ons are needed, and no third-party SDKs ship with the plugin. Google Maps is the only optional external integration - and the plugin degrades gracefully when its API key is absent.

The plugin is also white-label by design: the platform name, logo, email sender, brand colours, and container width are all overridable from Settings → CareerNest Settings → Branding / Appearance. The "CareerNest" string can disappear from the user-facing UI entirely.

Who is this for?

For Job Seekers

Build a public profile, upload your resume, apply for jobs, withdraw applications, and track every status change - all from a personal dashboard. Optional Google or LinkedIn sign-in.

For Employers

Register your company, await admin approval, post and manage jobs, review applications with full resume + cover-letter previews, and invite team members. One user can own multiple companies.

For Site Administrators

Approve employer signups, manage roles, customise email templates from a 3-panel visual editor, configure the job board, import/export CSVs, and white-label every visible string to match your brand.

System requirements

WordPress 6.0+, PHP 8.0+. A Google Maps API key is optional but strongly recommended - without it, the location filter on the public job board falls back to a literal substring match on the address string, which can hide jobs whose city or state isn't part of the saved address.

Installation & Setup

CareerNest is currently distributed as a manual install - it isn't yet listed on the WordPress.org plugin repository. There are two equivalent install paths.

Method 1 - ZIP upload via wp-admin

  1. From the WordPress admin dashboard, navigate to Plugins → Add New.
  2. Click Upload Plugin at the top of the page.
  3. Choose the career-nest.zip archive and click Install Now.
  4. Once installed, click Activate.

Method 2 - Manual file copy

  1. Extract career-nest.zip on your machine, or clone the repo directly.
  2. Upload the career-nest/ folder to wp-content/plugins/ via SFTP, SSH, or your hosting file manager. The full path should be wp-content/plugins/career-nest/careernest.php.
  3. In WordPress, open Plugins and click Activate next to CareerNest.

What happens on activation

Activating the plugin runs a one-time setup. You'll see the result of all of these the moment activation completes:

  • Creates 4 custom post types: job_listing, employer, applicant, job_application.
  • Creates 2 taxonomies attached to job_listing: job_category and job_type (Full-time, Part-time, Contract, Casual).
  • Creates 3 custom user roles: aes_admin, employer_team, applicant.
  • Grants plugin-management capabilities (manage_careernest, edit_jobs, edit_employers, ...) to administrator and aes_admin.
  • Auto-creates the managed front-end pages: /jobs/, /applicant-dashboard/, /employer-dashboard/, login, register-applicant, register-employer, register-employee, forgot-password, and reset-password.
  • Schedules the daily careernest_expire_jobs WP-Cron event (auto-expires jobs past their closing date).
  • Flushes the WordPress rewrite rules so the custom URLs work: /job/{slug}, /employers/{slug}, /applicants/{slug}.
After activation, see a 404?

If any CPT URL (like /job/test-job/) returns a 404 immediately after activation, visit Settings → Permalinks and click Save Changes with no changes. This force-flushes the rewrite rules. It's a one-time fix.

On deactivation

Deactivating the plugin unschedules the daily cron event but does not delete data. Your jobs, applicants, employers, applications, roles, and settings all remain intact. Reactivating restores everything.

On uninstall

If you remove the plugin entirely (via the WordPress Delete button on the Plugins screen), uninstall.php runs. It performs a thorough cleanup:

  • Clears all plugin options: careernest_options, careernest_pages, careernest_branding, careernest_appearance, careernest_email_templates, careernest_employer_dashboard, careernest_recent_activity.
  • Removes the managed CareerNest pages.
  • Clears the custom post type entries.
  • Removes the three custom roles and their capabilities.
  • Bulk-deletes any leftover OAuth state transients.
Uninstall is destructive

If you want to keep your data, choose Deactivate, not Delete. Uninstall removes everything CareerNest created. Always export your jobs and employers via CareerNest → Import / Export before uninstalling.

Quick Start (5-Minute Walkthrough)

The fastest path from zero to a working careers site. Run through these steps once after activation to see the plugin end-to-end.

  1. Activate the plugin.

    Once activated, you'll see a new top-level menu item - CareerNest - in the wp-admin sidebar, plus settings entries under Settings → CareerNest Settings.

  2. Add a Google Maps API key (optional but recommended).

    Visit Settings → CareerNest Settings → General and paste your Maps key into the Google Maps API Key field. This enables Places autocomplete on every address field and the radius search on the public job board.

  3. Set your branding.

    Switch to the Branding tab. Set your platform name (e.g. "AES Careers"), upload a logo, and configure the email-from name and address. These propagate everywhere user-facing.

  4. Register a test company.

    Sign out of wp-admin and visit /register-employer/ in an incognito window. Fill in the form. The company is created with pending status - it doesn't appear on the public job board yet.

  5. Approve the company.

    Sign back into wp-admin as administrator. Go to Employers, find your pending company, and approve it. The user receives an approval email (unless you've enabled Disable Emails) and their account is upgraded to employer_team role.

  6. Post your first job.

    Sign in as the employer. You'll land on the employer dashboard. Click Post a Job, fill the form (the address field will autocomplete if you set the Maps API key earlier), and publish.

  7. View it on the public job board.

    Open /jobs/ in another tab. Your job should appear with all filters working. Try the location radius slider with "Any distance" selected.

Skip the approval friction during testing

If you want to bypass the manual approval step while building or testing, enable Disable Emails under Settings → General first - that way no test emails escape to real inboxes while you're iterating.

Features Walkthrough

A guided tour of everything CareerNest can do, grouped by audience. Use the in-page screenshot placeholders as visual anchors - they're sized at 16:9 and ready to be swapped for real captures of your installation.

For Job Seekers (Applicants)

Applicants get a public-facing profile page (at /applicants/{slug}/) and a private dashboard at /applicant-dashboard/. The profile is fully self-managed via inline edit affordances - no need to switch into wp-admin.

The profile builder

Every section of the applicant profile can be edited inline by clicking the small pencil icon that appears on hover. Sections include:

  • Photo - drag-to-upload avatar, JPG/PNG/GIF, 5 MB cap, real-MIME validated.
  • Professional title - short headline shown under the name.
  • Bio / summary - rich text, sanitised with wp_kses_post().
  • Work experience - repeater. Each entry: job title, company, dates, description.
  • Education - repeater. Institution, qualification, dates.
  • Licenses & certifications - repeater.
  • Skills - comma-separated tags.
  • Location - Places autocomplete + lat/lng + Place ID (used by employers' search).
  • Work-type preferences - multi-select: Full-time, Part-time, Contract, Casual.
  • Right-to-work status - for jurisdictions that require disclosure.
  • Contact details - phone, email (display copy of the user's WP email).
  • Social / portfolio links - LinkedIn URL plus a repeater of custom links.

Resume management

The sidebar of the dashboard has a dedicated resume widget - upload, replace, or remove. Files are validated by real MIME (not the browser-supplied type, which is spoofable). Accepted: PDF, DOC, DOCX. Maximum size: 10 MB.

Applying to a job

From any job listing, click Apply. The apply form lets the applicant attach:

  • A resume (defaults to the one on file, but can be replaced for this application).
  • A free-text cover letter.
  • An optional cover-letter file (PDF/DOC).
  • Up to 5 additional documents (portfolio, references, work samples).

Bookmarks & status notifications

The bookmark icon on every job card is one-click toggle. Bookmarks appear in a dedicated tab on the applicant dashboard. Applicants receive an email any time their application status changes - submitted → under review → shortlisted → interview → offer → hired / rejected / withdrawn / archived.

Profile completeness widget

The dashboard sidebar shows a circular progress meter calculated from Profile_Helper::calculate_applicant_completeness(). Missing sections are listed below the meter, each with an "Edit" link that scrolls the user to the relevant section anchor on the public profile page.

For Employers / Hiring Companies

Employers get a public company page (/employers/{slug}/) and a private management dashboard at /employer-dashboard/. The dashboard is the operational hub: post jobs, review applications, manage team, update branding.

Registration & approval workflow

Anyone can submit a company at /register-employer/. The submission creates a pending employer post and a non-published user account in the employer_team role. Admins review the submission in wp-admin under Employers:

  • Approve - flips the employer to approved + publish, sends a welcome email with login link.
  • Decline - admin must enter a reason, which is included in the rejection email.
  • Request more info - admin writes a free-text message asking for clarification. The applicant gets an email with the message and a link back to the form to amend the submission.

Company profile builder

Every section is inline-editable from the public profile page (when signed in as an owner). Sections include: logo, tagline, mission, spotlight (a feature highlight), industry, company size, founded year, specialities, location (with Places autocomplete), website, contact email, phone, and the long-form "About" + "What we're interested in working on" blocks.

Posting a job

From the employer dashboard, Post a Job opens the job form. The form is organised into clearly-labelled sections:

  • Basics - title, category, type, location (with autocomplete + "Pick on map" modal), remote-position flag.
  • Vacancy Details - the main job description.
  • Essential Criteria - must-have requirements.
  • Desirable Criteria - nice-to-haves.
  • Responsibilities - bulleted day-to-day duties.
  • How to Apply - custom instructions to the candidate.
  • Salary - either a single value or a range; configurable per job.
  • Dates - opening date (defaults to publish date) and closing date (auto-expires past this).
  • External apply - optional URL. When enabled, "Apply" buttons link to that URL and the internal apply form is bypassed.
Pick from the autocomplete - don't manually edit

When entering an address, always select a result from the Places autocomplete dropdown. Typing or pasting an address manually clears the saved latitude/longitude (since the plugin can't be certain the new text still matches the previously-resolved place). Without coordinates, the job won't appear in radius-based search results.

Application management

The Applications tab on the dashboard lists every application received across all of the company's jobs. Each row shows the applicant name, the job they applied to, the submission date, and the current status. Click any row to see the full application - applicant profile snapshot, attached resume, cover letter (text + file), and any additional documents - all previewed inline. From the same view, update the status, add internal notes, and contact the applicant.

Team management

The Team tab lets a primary owner invite additional team members. Invitees receive an email and complete signup via a dedicated form. Once joined, they share access to the same company. The primary owner can also transfer ownership to another team member, which atomically swaps the _user_id primary-owner pointer.

Multi-owner / multi-company

One user can be a co-owner of any number of companies, and one company can have multiple owners. When a user is linked to more than one company, an active-company switcher pill appears on the dashboard hero - clicking it lists all linked companies and lets them swap which one they're managing right now.

Position filled

Each job has a "Position filled" toggle. Once set, the job is removed from the public board automatically (without needing to set the closing date in the past). Useful when a vacancy fills mid-cycle.

Dual-Role Support 1.15.0+

A single WordPress user can hold any combination of CareerNest roles plus standard WP roles - for example, applicant + employer_team + administrator all on one account. The plugin provides a coherent UX for these dual-role users: a mode switcher, upgrade-mode forms, and discovery affordances that nudge a user from one role into the other.

Mode switcher

When a user has both applicant and employer_team roles, a "Viewing as: Applicant ▼" pill appears in both dashboard heroes. Clicking it switches modes - under the hood, an AJAX call (cn_switch_dashboard_mode) writes _active_dashboard_mode user meta and reloads to the other dashboard. The login-redirect honours this preference, so a user who last switched to "Employer" lands back on the employer dashboard the next time they sign in.

Upgrade-mode register forms

When a logged-in applicant visits /register-employer/, the form detects the existing account and renders in upgrade mode: email is locked (no re-entry), password is hidden (the user is already authenticated), social buttons are hidden, and the submission adds the employer_team role to the existing user rather than creating a new account. The reverse works identically for employer-to-applicant upgrades.

Discovery affordances

Users only become aware of dual-role if we tell them. Three nudges:

  • Dashboard sidebar cards. "Apply for jobs too →" on the employer dashboard (if no applicant role). "Hiring? Post a job →" on the applicant dashboard (if no employer role).
  • Apply-button intercept. If an employer-only user clicks Apply on a job listing, an intercept card appears: "You're signed in as an employer. Add an applicant profile to apply →" - clicking takes them to the upgrade-mode register form.
  • OAuth intent. Signing in via Google/LinkedIn from one of the register pages passes an intent=add_employer (or add_applicant) parameter through the OAuth flow, so the post-login redirect lands on the right upgrade flow rather than the default dashboard.
Granting dual roles from admin

An administrator can assign any combination of CN-managed roles to a user via Users → Manage Roles. This is the canonical role-management UI - see Administrator Tools below.

Authentication Options

CareerNest supports four authentication paths, all of which can coexist on the same site:

Standard email + password

The default. Sign-up via /register-applicant/ or /register-employer/. Sign-in via /login/ or the [careernest_login] shortcode placed on any page.

Google OAuth

Enable under Settings → General → Social Login. You'll need a Google Cloud Console project with OAuth 2.0 credentials configured. The plugin handles three OAuth paths automatically:

  • New user, picks Google - lands on a role-chooser page after Google authentication, then completes signup.
  • Existing user, signs in via Google - if the Google email matches a WP user, accounts auto-link.
  • Logged-in user, clicks "Connect Google" from the Linked Accounts panel on their dashboard - links Google to their existing account for future passwordless sign-in.

LinkedIn OAuth

Same architecture as Google. Configure under Settings → General → Social Login. LinkedIn's permissions model is more restrictive than Google's - only basic profile + email scopes are requested.

Two-Factor Authentication

If a third-party 2FA plugin is installed (e.g. the official Two Factor plugin or Wordfence Login Security), the Linked Accounts panel surfaces a "Set up 2FA" link that points to the standard wp-admin profile route. Enable the row's visibility under Settings → Social Login → Show 2FA in Linked Accounts. The plugin doesn't ship its own 2FA implementation - it integrates with whatever you already use.

Password reset / account deletion

The /forgot-password/ and /reset-password/ pages are auto-created on activation and use the standard WP password-reset email flow (routed through Mailer::* for the disable-emails-during-dev override). Account deletion is request-based - users submit a deletion request from their dashboard, an admin reviews and approves it in wp-admin, and only then is the WP user actually deleted (GDPR Article 17 friendly).

Personal Data Exporter / Eraser

The plugin registers handlers with the standard WP Personal Data Exporter and Eraser (under Tools → Export Personal Data / Erase Personal Data). OAuth-linked metadata (provider ID, email at provider, avatar URL, link timestamp) is included in both the export bundle and the erase pass.

Job Board & Search

The public job board lives at /jobs/ (the page slug is configurable under Settings → Pages). It's a single-page experience powered by AJAX - filter changes refresh the result set without a full page reload.

Available filters

  • Keyword - searches job title + description content.
  • Category - taxonomy filter, multi-select.
  • Job type - Full-time, Part-time, Contract, Casual.
  • Location - Places autocomplete on the input. Hidden lat/lng power the radius search.
  • Radius - slider, 0 to 100 km. 0 = "Any distance" shows every job sorted by distance from the searched location (no geographic cut-off).
  • Employer - filter to a specific company.
  • Salary range - dual-handle slider.
  • Date posted - last 24 h, week, month, all time.

Distance badges

When the user provides a location, every result card gets a distance badge ("5 km from your search", "320 km away") computed via the Haversine formula on the server. Results are sorted by distance ascending unless another sort is selected.

Bookmarks, view tracking, external-click tracking

Logged-in users can bookmark jobs from the result card or the single-job page. View counts and external-click counts (for jobs configured with an external apply URL) are tracked transparently. Aggregated stats are visible to the owning employer on the employer dashboard.

Configurable layout

Under Settings → General → Job Listing Filters you can:

  • Move the filter column from sidebar to top.
  • Choose 1, 2, or 3 columns for result cards.
  • Enable a map-view toggle (renders results on an embedded Google Map).
  • Drag-reorder which filters appear, and toggle individual filters' visibility.

Administrator Tools

Site administrators (anyone holding manage_options) get a complete admin surface in wp-admin. Power users with the custom aes_admin role get access to everything CareerNest-related without being able to manage plugins, themes, or the WordPress core.

Users → Manage Roles 1.17.0

A dedicated page (under the standard Users menu) for granting and revoking CN-managed roles. Built as an extended WP_List_Table:

  • Every user is a row, every CN-managed role is a column.
  • Each cell is a click-to-toggle button - ✓ (granted), ✗ (not granted), spinner (in progress).
  • Search filter (display name + email), role filter dropdown (All / Has applicant / Has employer / Dual-role / None), pagination (20 per page).
  • Last-administrator guard - the page refuses to remove administrator from the last admin on the site.
  • Self-edit guard - your own row's cells are disabled (preventing accidental self-demotion).
  • Granting applicant for the first time auto-creates an applicant CPT stub so the user has data to render on their dashboard.

Defensive role-wipe protection 1.17.2

WordPress's standard user-edit form calls $user->set_role($primary) on every save - even when the role dropdown wasn't touched - which destructively wipes all roles and sets exactly one. The plugin hooks set_user_role at priority 999 to restore any CN-managed roles that the wipe removed. This means roles granted via Manage Roles survive ordinary user-profile edits.

Linked Employers panel

On the standard user-edit screen (Users → All Users → Edit), users in the employer_team role see a "Linked Employers" panel. It lists every company they're currently linked to and provides add/remove controls. Behind the scenes, this uses the same AJAX endpoints as the employer-side meta box.

Per-CPT meta boxes

Every custom post type has tailored meta boxes in its edit screen:

  • job_listing - Job Details (location, salary, dates, content sections, employer assignment).
  • employer - Employer Details, Company Owners, Team Management, Employer Team Members.
  • applicant - Applicant Details, Skills, Work Preferences.
  • job_application - Application Details (with resume + cover letter + extra documents previews), Application Status.

Custom admin list columns + quick filters

The admin list tables for jobs, employers, applicants, and applications have custom columns (logo, location, status, employer name, etc.) and quick-filter dropdowns above the table.

Email template visual editor 1.9.0

Under Settings → CareerNest Settings → Email Templates: a 3-panel UI - left sidebar with the 15 templates, centre column with a TinyMCE editor, right column with a live-preview iframe. Saves are AJAX per-template. Available actions: Save, Reset to default, Send test email (to the logged-in admin's address).

Import / Export

Under CareerNest → Import / Export: download a CSV template, populate it, upload it. Supported for both jobs and employers. The import is transactional - if any row fails validation, no rows are committed.

Dev-mode email kill switch

Under Settings → General → Disable Emails. When enabled, every email sent by the plugin is logged to error_log instead of being delivered. A persistent warning banner appears at the top of wp-admin to remind you it's on. Essential for testing - toggle it off before going live.

Debug: Show Coordinates

Also under Settings → General. When enabled, every job card on the public board displays the saved latitude/longitude. Useful for diagnosing the most common location-search bug - jobs saved without coordinates.

White-Label Branding

CareerNest is built to disappear behind your brand. Every visible string and colour can be overridden from Settings.

Platform identity

  • Platform name - replaces "CareerNest" everywhere user-facing. Set under Branding → Platform Name.
  • Platform logo - uploaded via the standard WP media library. Used in headers, emails, and the login page.

Email branding

  • Email From Name - sender display name. Defaults to "The {Platform Name} Team".
  • Email From Address - sender address. Defaults to the WP admin email.
  • Support Email - surfaced inside email bodies via the {{contact_email}} placeholder. Defaults to WP admin email.

The wp_mail_from / wp_mail_from_name filters are scoped - they only apply during Mailer::* calls (guarded by a static flag), so any other plugin sending emails on the same site is unaffected.

Brand colours via CSS variables

Set under Appearance:

  • Primary button colour (and an auto-derived hover state)
  • Secondary button colour
  • Primary text colour (headings, labels)
  • Secondary text colour (descriptions, meta)
  • Success / warning / danger badge colours
  • Container width (max-width override for the layout grid)

These emit as CSS custom properties on :root across every front-end page (not just CareerNest pages), so your theme can pick them up via var(--cn-primary-btn, #FF8200) if you want to share the same brand tokens.

Auto-managed options (not user-editable)

Two additional options are populated automatically and shouldn't be edited by hand:

  • careernest_pages - created on activation. A page-slug → page-ID map for every managed page (jobs, login, applicant-dashboard, employer-dashboard, register pages, password-reset pages). The plugin uses this map in its template_include filter to swap in the right template per page.

If you add a new option, update uninstall.php

The uninstall script explicitly deletes the seven known option keys above. Adding a new careernest_* option without also adding it to uninstall.php will leave orphan data after deletion.

Everything theme developers and plugin extenders need to integrate with CareerNest - shortcodes, hooks, filters, helper functions, helper classes, and an overview of the AJAX surface.

Five shortcodes are registered. Drop any of them into a page, post, widget, or template via do_shortcode().

TagRenders

8.2 Action Hooks

Three plugin-defined actions extension code can listen on:

// Fired when an applicant withdraws an application.
do_action('careernest_application_withdrawn', int $application_id);

// Fired when an employer updates application status.
do_action('careernest_application_status_changed', int $app_id, string $old_status, string $new_status);

// Daily cron tick - auto-expires jobs past _closing_date.
do_action('careernest_expire_jobs');

Listen for these from any plugin or theme via add_action(). Common use case: integrate with a CRM by posting to a webhook whenever an application status changes.

8.3 Template-Override Filters

Six filters let a theme override CareerNest's built-in templates without forking the plugin:

// Override individual CPT templates.
apply_filters('careernest_single_job_template',      $template, $original);
apply_filters('careernest_single_employer_template', $template, $original);
apply_filters('careernest_single_applicant_template',$template, $original);

// Override managed-page templates (jobs, login, dashboards, register, etc.).
apply_filters('careernest_page_templates', $template_map);
apply_filters("careernest_page_template_{$page_slug}", $template);
apply_filters('careernest_page_template', $template, $page_slug);

Return the absolute path to your replacement template file. The plugin's default templates live in career-nest/templates/ - copy any of them into your theme as a starting point.

8.4 PHP Helper Functions

Three branding helpers are available globally (defined in includes/branding-helpers.php):

cn_get_platform_name();      // "CareerNest" or branded override
cn_get_email_from_name();    // Email sender display name
cn_get_email_from_address(); // Email sender address

Use these everywhere user-facing strings refer to the platform - they automatically honour the white-label settings.

8.5 Public Helper Classes

Two static helper classes encapsulate the most important plugin logic. Always use these rather than reading meta directly.

Ownership & access checks

// Always use these - never read _user_id / _owner_ids directly.
CareerNest\Team_Manager::is_owner(int $user_id, int $employer_id): bool;
CareerNest\Team_Manager::user_can_manage_employer(int $user_id, int $employer_id): bool;

is_owner() checks both the legacy scalar _user_id (primary owner) AND the modern _owner_ids array (co-owners). user_can_manage_employer() is broader - it additionally walks the user-side _employer_id scalar and _employer_ids array. Use the former for "is this user specifically an owner?" and the latter for "can this user act on this employer?".

Dual-role awareness

// Returns 'applicant' or 'employer' - honours the user's last-chosen mode.
CareerNest\Profile_Helper::get_active_dashboard_mode(int $user_id): string;

// True if the user holds both applicant AND employer_team roles.
CareerNest\Profile_Helper::user_has_dual_roles(int $user_id): bool;

// Both return ['percentage' => int, 'missing' => string[]] for the
// dashboard completeness widget.
CareerNest\Profile_Helper::calculate_employer_completeness(int $employer_id): array;
CareerNest\Profile_Helper::calculate_applicant_completeness(int $applicant_id): array;

Mailer (always go through this)

// Send a registered template. $vars are merged with site defaults.
CareerNest\Email\Mailer::send(string $template_key, string $to, array $vars);

// Plain-text email, no template.
CareerNest\Email\Mailer::send_plain(string $to, string $subject, string $message);

// Notification to the site admin (uses WP admin email).
CareerNest\Email\Mailer::send_admin_notification(string $subject, string $message);

Never call wp_mail() directly from inside CN-related code - it bypasses the dev-mode disable flag and the scoped from-address override.

8.6 AJAX & admin_post Endpoints

The plugin exposes 35+ AJAX actions (half legacy careernest_*, half modern cn_*) and 13 admin_post_* handlers. They power the approval workflows, role toggles, dashboard mode switching, file uploads, application status updates, and CSV import/export. A full table is maintained in the plugin's internal CLAUDE.md.

Notable AJAX endpoints

  • careernest_filter_jobs - the public job-board filter (with nopriv variant).
  • cn_toggle_user_role - the Manage Roles page's toggle handler.
  • cn_switch_dashboard_mode - dual-role mode switcher.
  • cn_add_employer_owner / cn_remove_employer_owner - multi-owner management.
  • cn_switch_employer - active-company switcher for users owning multiple.
  • cn_create_job / cn_update_job / cn_delete_job - front-end job CRUD.
  • cn_update_app_status - application status update (fires careernest_application_status_changed).
  • cn_upload_avatar / cn_upload_resume - applicant file uploads.

Notable admin_post endpoints

  • cn_approve_employer / cn_decline_employer / cn_request_info - employer approval workflow.
  • cn_approve_employee / cn_decline_employee - team-member approval workflow.
  • cn_approve_deletion / cn_decline_deletion - GDPR deletion-request workflow.
  • cn_export_jobs / cn_import_jobs / cn_export_employers / cn_import_employers - CSV import/export.

When adding a new AJAX endpoint, register the wp_ajax_* hook unconditionally (and wp_ajax_nopriv_* if it's a public endpoint). Capability and nonce checks go inside the handler. The same rule applies to admin_post_* handlers - register in hooks() regardless of screen, since admin-post.php requests never trigger admin_menu or current_screen.

A note on white-labelling

CareerNest is intentionally white-label. The plugin name, logo, colours, and email sender details that appear to your end users can all be overridden from Settings → CareerNest Settings → Branding / Appearance. Nothing in the customer-facing experience needs to mention "CareerNest" - your platform can carry your brand entirely.

If you're documenting your own deployment for your users, feel free to fork this manual and rebrand it. Everything below the surface is yours to relabel.

Thanks for using CareerNest - and good luck with your careers platform.
